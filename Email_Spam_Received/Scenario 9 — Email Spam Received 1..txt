Scenario 9 — Email Spam Received 1.1
The scenario begins at S0_Spam_Received_ABN, the initial situation. At this state, two events enter simultaneously — E1_Click_Link_ABN and E2_Open_Attachment — and the threat T1_Email_Spam is also present. From S0, two paths immediately diverge based on which event the user acts upon.
The first path is the attachment path, triggered by E2_Open_Attachment. Action a01_Opening_ABN leads the system to S1_Infected_Attachment_Opened_ABN. At S1, event E3_Run_Antivirus enters. The item Infected_Email_Attachment is present at this state, and it is described by two further items — Program and Viral_Code — both linked to the infected attachment. From S1, two paths diverge. The first is the corrective path: action a12_Running_Antivirus_COR leads to S2_Antivirus_Active, at which point the item Antivirus_Software enters scope. From S2, action a26_Resulting_In leads to the normal terminal state S6_Normal_State. The second path from S1 is the malicious path: if antivirus is not run, action a17_Infecting_ABN leads to S7_Infected_With_Virus_ABN, at which point the viral code infects the host. From S7, action a79_Overtaking_ABN leads to the abnormal terminal state S9_Machine_Overtaken_By_Hacker_ABN.
The second path from S0 is the malicious link path, triggered by E1_Click_Link_ABN. Action a03_Clicking_Link_ABN leads the system to S3_Malicious_Link_Clicked_ABN. At S3, event E4_Run_Antivirus enters. The items URL and Malicious_JavaScript both enter scope at this state — the URL carries the malicious script payload. From S3, three paths diverge.
The first path from S3 is again corrective: action a32_Running_Antivirus_COR leads to S2_Antivirus_Active, the same shared corrective state reached from the attachment path. The item Antivirus_Software enters scope, and action a26_Resulting_In again leads to the normal terminal state S6_Normal_State.
The second path from S3 is the redirect path: action a34_Redirecting_ABN leads to S4_Redirected_To_Hackers_Website_ABN. At S4, event E5_Credentials_Stolen_By_Website_ABN enters. Action a45_Extorting_ABN then leads to the abnormal terminal state S5_Vital_Data_Extorted_ABN, representing successful credential theft via the hacker's spoofed website.
The third path from S3 is the JavaScript infection path: action a38_Infecting_ABN leads to S8_Infected_With_JavaScript_ABN, at which point the malicious script begins executing on the host. From S8, action a89_Overtaking_ABN leads to the same abnormal terminal state S9_Machine_Overtaken_By_Hacker_ABN that is also reachable from S7 on the attachment path.
The scenario therefore induces five named execution paths. ρ_attachment-corrective (S0 → S1 → S2 → S6) and ρ_link-corrective (S0 → S3 → S2 → S6) both converge at S2 before reaching the corrective terminal. ρ_virus-takeover (S0 → S1 → S7 → S9) and ρ_javascript-takeover (S0 → S3 → S8 → S9) both converge at S9 as the criminal terminal, approached from two structurally independent infection vectors. ρ_credential-theft (S0 → S3 → S4 → S5) constitutes the third criminal terminal path, reached exclusively via the link path and the redirect mechanism. The two criminal convergence paths at S9 are structurally symmetric in their terminal valuation: both result in machine takeover and produce the same downstream check-level valuations from S9 onward, regardless of whether the host was compromised via viral code or malicious JavaScript.
