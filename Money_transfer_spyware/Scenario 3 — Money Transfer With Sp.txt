Scenario 3 — Money Transfer With Spyware 2.0
The scenario begins at S0_Browser_Started, the initial situation. From S0, two paths immediately diverge. The first is an abnormal termination: action a021_Refusing_Connection_ABN fires and the system transitions directly to S17_Site_Maintenance_ABN, an abnormal terminal state representing a refused connection at the entry point. The second path is the main flow: action a01_Requesting leads to S1_Request_For_Connecting.
At S1_Request_For_Connecting, event E2_Network_Problem_ABN enters. The system transitions to S2_Connected via action a12_Connecting.
At S2_Connected, event E1_Click_Link enters. The system enters a self-loop through action a22_Checking_If_Website_Live, cycling back to S2 until the site is confirmed live. Once resolved, S2 branches into three outgoing paths. The first leads to S18_Disconnected via action a218_Refusing_Login, an abnormal terminal state representing a login refusal. The second leads to S3_Credentials_Sent via action a23_Logging_In. The third — the malicious sub-path — leads to S12_Malicious_Link_Clicked_ABN via action a212_Opening_ABN.
At S3_Credentials_Sent, events E4_Credentials_Rejected and E5_Verify_Credentials both enter. The state branches into two paths. The first leads to the abnormal terminal state S19_Invalid_Credentials via action a319_Sending_Credentials, triggered when credentials are rejected. The second path leads to S4_Logged_In via action a34_Verifying_Credentials.
From S4_Logged_In, the system proceeds sequentially along the legitimate path. S4 leads to S5_Operation_Chosen via a45_Choosing_Operation, then to S6_Balance_Requested via a56_Requesting_Balance. At S6, event E6_Return_Balance enters, and the system transitions to S7_Balance_Shown via a67_Showing_Balance.
At S7_Balance_Shown, two paths diverge. The first leads to S21_Insufficient_Amount_ABN via action a72_Declining_ABN, representing an operation declined due to insufficient funds. From S21, action a212_Cancelling_Operation_ABN leads to the abnormal terminal state S22_Unauthorized_Operation_ABN. The second path from S7 leads to S8_Money_Send via action a78_Executing_Operation, triggered when event E7_Transfer enters S8.
At S8_Money_Send, the system again branches. Action a82_Running_Analytics leads to S9_Transaction_Successful, then action a89_Clearing_Transaction leads to S9, and from there action a91_Logging_Out leads to the normal terminal state S10_Logged_Out. Alternatively, if event E10_User_Logout fires during clearing, the system transitions to the abnormal terminal state S20_Transaction_Cancelled_ABN via a89_Clearing_Transaction.
Returning now to the malicious sub-path that branched from S2. At S12_Malicious_Link_Clicked_ABN, action a113_Infecting_ABN leads to S13_Infected_With_Malware_ABN, at which point the threat T1_Spyware appears in this situation. From S13, action a114_Capturing_ABN leads to S14_Keystrokes_Captured_ABN, and from S14, action a115_Extorting_ABN leads to S15_Vital_Data_Extorted_ABN.
At S15_Vital_Data_Extorted_ABN, event E8_Run_Antivirus enters and the state branches into two corrective paths. The first path leads back into the legitimate transaction flow: action a158_Authenticating_ABN re-enters the system at S8_Money_Send, rejoining the prior paths that lead to either S20_Transaction_Cancelled_ABN or S10_Logged_Out as described above. The second path is the full corrective remediation path: action a156_Removing_Malware_From_COR leads to S16_Normal_State, at which point event E9_User_Logout enters, and action a161_Logging_Out transitions the system directly to the normal terminal state S10_Logged_Out.
The scenario therefore induces the following named execution paths: ρ_refused (S0 → S17), ρ_disconnected (S0 → S1 → S2 → S18), ρ_invalid (S0 → S1 → S2 → S3 → S19), ρ_insufficient (S0 → S1 → S2 → S3 → S4 → S5 → S6 → S7 → S21 → S22), ρ_cancelled (S0 → S1 → S2 → S3 → S4 → S5 → S6 → S7 → S8 → S20), ρ_legitimate (S0 → S1 → S2 → S3 → S4 → S5 → S6 → S7 → S8 → S9 → S10), ρ_remediation (S0 → S1 → S2 → S12 → S13 → S14 → S15 → S16 → S10), and ρ_criminal-rejoin (S0 → S1 → S2 → S12 → S13 → S14 → S15 → S8 → S20 or S10).
